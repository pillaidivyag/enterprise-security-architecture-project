# Ubuntu Hardening

## Purpose

This document records the operating system hardening activities performed before deploying the application stack.

---

## Hardening Activities

The following security measures were implemented during the initial system preparation:

- Installed the latest security updates.
- Removed unnecessary packages.
- Used a standard user account with `sudo` for administration.
- Verified network connectivity.
- Prepared the system for additional security controls.

Future hardening activities include:

- Configure UFW Firewall.
- Configure Fail2Ban.
- Harden SSH.
- Configure automatic security updates.
- Review file permissions.
- Enable security logging.

---

## Outcome

The operating system has been updated and prepared for secure application deployment. Additional hardening will be completed after all application components have been installed.
