# Evidence

## Purpose

This folder contains implementation evidence collected during the deployment, hardening, and validation of the Enterprise Security Architecture & Secure Nextcloud Deployment project.

The artefacts demonstrate the implementation of security controls, validation of the deployed environment, and remediation of security findings identified during the project.

## Evidence Inventory

| File                              | Description                                                                                         |
| --------------------------------- | --------------------------------------------------------------------------------------------------- |
| `Nginx-Welcome-Page.png`          | Verification of successful Nginx web server deployment before Nextcloud installation.               |
| `Nextcloud-setup-page.png`        | Initial Nextcloud installation and configuration screen.                                            |
| `Security-Warnings.png`           | Initial security and setup warnings identified after deployment.                                    |
| `Security-Warnings.drawio`        | Working diagram used to document and track remediation of identified security warnings.             |
| `Nextcloud-Security-Overview.png` | Administrative security overview used during security validation.                                   |
| `resolved warnings.png`           | Security overview after implementing recommended security controls and resolving identified issues. |
| `TLS-Certificate-Details.png`     | Verification of HTTPS and TLS certificate configuration.                                            |
| `UFW-HTTPS-Rule.png`              | Firewall configuration allowing secure HTTPS communication.                                         |
| `Least-Privilege.png`             | Evidence supporting implementation of least privilege principles.                                   |
| `Least-Privilege.md`              | Documentation describing least privilege implementation and validation.                             |
| `Errors-in-the-log.png`           | Evidence collected during investigation and validation of application log errors.                   |

## Evidence Usage

The evidence contained within this directory supports:

* Security Architecture validation
* Security Control Validation Report
* Risk Assessment
* Threat Modelling
* Final Security Assessment Report

## Note

Where applicable, sensitive information has been removed or redacted before publication. These artefacts are intended solely to demonstrate the implementation and validation of security controls within this portfolio project.
