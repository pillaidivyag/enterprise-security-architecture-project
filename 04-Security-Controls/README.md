## Overview

This phase defines the security controls recommended for the proposed Nextcloud Enterprise deployment. The controls have been selected to address the threats identified during the threat modelling exercise and reduce the business risks documented in the Risk Assessment.

The objective is to ensure the solution is secure by design while supporting the organisation's operational and business requirements.

The recommendations are based on recognised security principles including Defence in Depth, Least Privilege, Zero Trust and Secure by Default.

---

## Security Control Categories

The recommended controls are organised into the following areas:

- Identity and Access Management
- Data Protection
- Network Security
- Application Security
- Logging and Monitoring
- Availability and Resilience

Each control category maps directly to risks identified during the previous phases of the assessment.

## Security Design Principles

The following principles have guided the recommended security controls.

- Defence in Depth
- Zero Trust
- Least Privilege
- Secure by Design
- Separation of Duties
- Continuous Monitoring
- Risk-Based Security

---

## Objectives

The recommended controls aim to:

- Protect confidential business information
- Prevent unauthorised access
- Detect malicious activity
- Reduce the attack surface
- Support business continuity
- Improve incident response capability
- Maintain regulatory compliance
